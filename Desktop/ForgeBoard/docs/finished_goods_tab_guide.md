# Finished Goods Tab Guide

This note explains what the `Finished Goods` tab in ForgeBoard shows and how each number is calculated.

## Purpose

The `Finished Goods` tab helps the planner inspect one finished good at a time.

It combines:

- a finished-good selector
- four headline FG metrics
- a fulfillment summary card
- a blocker-reason table
- limiting component badges
- blocking component badges
- a shortage table for the selected FG

The tab is driven by one row from `result["analyses"]` for the selected finished good.

## Data Source

The tab uses:

- `result["analyses"]`: finished-good level analysis generated by the planning engine
- `fg["shortages"]`: shortage rows for the selected FG
- `fg["limiting_components"]`: strict mathematical bottlenecks
- `fg["blocking_components"]`: planner-facing blocker shortlist

## Client-Friendly Explanation

`The Finished Goods tab lets the planner open one product at a time and understand its production situation clearly. It shows how much still needs to be made, how much can be made with current stock, which materials are stopping production, and which shortages matter most for that specific product.`

### Simple explanation of the selector

- `Choose finished good`: pick the product you want to inspect in detail

### Simple explanation of the top numbers

- `Net Demand`: how much of this finished good still needs to be produced
- `Max Producible`: the maximum quantity that can be made right now with current materials
- `Recommended Build`: what ForgeBoard suggests building now for this item
- `Coverage`: what percentage of open demand can be covered immediately

### Simple explanation of the fulfillment summary

`This card tells the planner clearly whether the current demand can be fully met. If not, it shows how many units can still be built now, how much demand remains unmet, and what percentage of the demand can be fulfilled immediately.`

### Simple explanation of the blocker-reason table

`This table answers a practical planner question: which raw materials are stopping this finished good from being fully made right now.`

It shows:

- the limiting material name
- how much of that material is needed for one FG
- how much is available now
- how many FG units that material alone would allow
- how much quantity is required and short

### Simple explanation of the component badges

- `Limiting components`: the exact materials that mathematically cap production
- `Blocking components`: the practical blocker shortlist the planner should focus on first

### Simple explanation of the shortage table

`This table shows the full shortage list for the selected finished good, sorted from the largest shortage downward, and explains how large each material gap is.`

- `Component`: missing or constrained material
- `Qty / FG`: how much of that material is needed for one finished good unit
- `Required`: total quantity needed for current demand
- `Available`: current stock available
- `Shortage`: missing quantity

### Simple explanation of each Top Shortage Lines column

- `Component`: the material or part causing pressure for this finished good
- `Qty / FG`: quantity of that material needed to produce one unit of the selected FG
- `Required`: total quantity needed for the full current net demand
- `Available`: stock currently available for that material
- `Shortage`: how much more material is needed to fully support the selected FG demand

Simple client example:

`If one finished good needs 2 units of a component, demand is 1000, and stock available is 500, then required quantity is 2000 and shortage is 1500.`

## Tab Layout

The tab renders one selected FG from the scenario analysis and shows:

- `Net Demand`
- `Max Producible`
- `Recommended Build`
- `Coverage`
- `Fulfillment summary`
- `Why this FG cannot be fully made`
- `Limiting components`
- `Blocking components`
- `Top shortage lines`

## Top Metrics

### Net Demand

Formula:

```text
Net Demand = max((Demand Qty x demand_multiplier) - FG On-hand Qty, 0)
```

Meaning:

Open quantity that still needs production after subtracting finished-good stock already available.

### Max Producible

Formula:

```text
possible units from a component = available_qty / qty_per_fg

Max Producible = floor(min(possible units across all required components))
```

Meaning:

Maximum whole units that can be produced for this FG using current inventory.

### Recommended Build

Formula:

```text
Recommended Build = min(Max Producible, floor(Net Demand))
```

Meaning:

The build quantity ForgeBoard recommends for this FG before final shared-inventory allocation across all FGs.

### Coverage

Formula:

```text
Coverage = Recommended Build / floor(Net Demand)
```

Special case:

```text
If Net Demand == 0, Coverage = 1.0
```

Meaning:

Share of current net demand that can be satisfied immediately.

## Fulfillment Summary

This card is the plain-language interpretation of the FG feasibility numbers.

It answers:

- can this finished good be fully supplied right now
- if not, how many units can still be built now
- how many units remain unmet
- what percentage of demand can be fulfilled immediately

Displayed values:

- `Build now`: current `Recommended Build`
- `Unmet demand`: `Net Demand - Recommended Build`
- `Fulfillment`: current coverage percentage

Possible states:

- `Demand already covered`: there is no open net demand
- `Full fulfillment possible`: all open demand can be produced now
- `Partial fulfillment`: only part of the demand can be produced now
- `Currently blocked`: no units can be produced now with current stock

## Why This FG Cannot Be Fully Made

This table is the explicit raw-material blocker view for the selected FG.

It is built from the FG's limiting components and their matching shortage rows.

Displayed columns:

- `Component`: limiting material name
- `Qty / FG`: quantity needed for one FG
- `Available`: current stock for that material
- `Possible FG Units`: how many FG units this material alone would allow
- `Required`: total quantity needed for current net demand
- `Shortage`: missing quantity for full demand coverage
- `Why blocking`: plain-language blocker type

Blocker labels:

- `Zero stock hard stop`: available quantity is zero, so this material alone prevents any build
- `Lowest build limit`: some stock exists, but this material still sets the lowest production ceiling

## Limiting Components

Formula logic:

```text
For each component:
  possible units = available_qty / qty_per_fg

limiting value = minimum possible units across all components

Limiting components = all components whose possible units equal the limiting value
```

Meaning:

These are the exact components that mathematically cap the FG's production quantity.

Why multiple components can appear:

If two or more components produce the same lowest production limit, all of them are shown as limiting components.

Worked example:

Suppose one finished good needs:

- `BASY0205`: `1` per FG, available `0`
- `PACKTC1D115`: `2` per FG, available `0`
- `COPS0420`: `4` per FG, available `500`
- `COPS0095`: `3` per FG, available `700`

Then:

```text
possible units from BASY0205 = 0 / 1 = 0
possible units from PACKTC1D115 = 0 / 2 = 0
possible units from COPS0420 = 500 / 4 = 125
possible units from COPS0095 = 700 / 3 = 233
```

So the limiting components are:

- `BASY0205`
- `PACKTC1D115`

because both cap production at `0` units.

## Blocking Components

Formula logic:

```text
Blocking components =
  limiting components
  + largest shortage components not already included
  until the list reaches 5 items
```

Meaning:

This is the practical planner-facing blocker shortlist for the selected FG.

Difference from limiting components:

- `Limiting components` are strict mathematical bottlenecks.
- `Blocking components` are a short operational shortlist that starts from those bottlenecks and then adds major shortage lines.

Important note:

`The Blocking components badge row is intentionally kept short for readability. Use the Why this FG cannot be fully made table for the detailed limiting-material explanation.`

Worked example:

Using the same FG example above, assume the FG-specific shortage quantities are:

- `BASY0205` shortage = `1000`
- `PACKTC1D115` shortage = `2000`
- `COPS0420` shortage = `3500`
- `COPS0095` shortage = `2500`

ForgeBoard first keeps the limiting components:

- `BASY0205`
- `PACKTC1D115`

Then it adds the biggest shortage lines not already included:

- `COPS0420`
- `COPS0095`

So the result becomes:

- `Limiting components`: `BASY0205`, `PACKTC1D115`
- `Blocking components`: `BASY0205`, `PACKTC1D115`, `COPS0420`, `COPS0095`

This shows the difference clearly:

- `Limiting components` answer: `what mathematically stops the build`
- `Blocking components` answer: `what the planner should look at first`

## Top Shortage Lines Table

The shortage table is built from the selected FG's full `shortages` list.

Current UI behavior:

`The table is not capped. It shows the full shortage list for the selected finished good.`

Important note:

`A component can still be a limiting or blocking component even if it does not appear near the top of this table. This table is sorted by shortage quantity, while limiting logic is based on the lowest possible build quantity.`

### Component

Meaning:

The raw material, part, or subassembly under shortage pressure for this FG.

### Qty / FG

Formula:

```text
Qty / FG = component quantity required for one FG unit
```

Meaning:

How much of that component is consumed to produce one finished-good unit.

### Required

Formula:

```text
Required = Net Demand x Qty / FG
```

Meaning:

Total component quantity needed to cover the current net demand for this FG.

### Available

Formula:

```text
Available = current inventory for that component
```

Meaning:

Stock currently available before final cross-FG allocation.

### Shortage

Formula:

```text
Shortage = max(Required - Available, 0)
```

Meaning:

Material gap preventing this FG from reaching full demand coverage.

Sort order:

```text
Rows are sorted by Shortage descending
```

So the largest shortages appear first.

## Important Interpretation Note

The `Finished Goods` tab is an FG-level feasibility view, not the final multi-FG allocation result.

That means:

- the tab explains what is feasible for the selected FG on its own
- the final production plan may differ after higher-priority FGs consume shared inventory

So this tab is best used for diagnosis, while the production plan is best used for final execution order.

## No-BOM Edge Case

If ForgeBoard cannot find a BOM for the selected FG:

- `Max Producible = 0`
- `Recommended Build = 0`
- `Coverage = 0`
- `Limiting components = []`
- `Blocking components = ["NO_BOM_FOUND"]`
- shortage lines are empty

Meaning:

The system cannot calculate component feasibility for that FG until BOM data exists.

## Practical Reading Order

For planners, the fastest reading order is:

1. Select the FG you want to inspect.
2. Check `Net Demand` and `Coverage`.
3. Check `Max Producible` versus `Recommended Build`.
4. Review `Limiting components` to see the true bottlenecks.
5. Review `Blocking components` for the practical action list.
6. Review `Why this FG cannot be fully made` for exact raw-material blocker details.
7. Review `Top shortage lines` to quantify the material gaps.
